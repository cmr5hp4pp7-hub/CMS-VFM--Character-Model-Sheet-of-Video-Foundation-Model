CMS-VFM (Visual Standardization Management "Example Based on readme.md Content")

CMS-VFM is an engineering specification for video generation, ensuring consistency of characters, props, and visuals in long narratives, multiple shots, and cross-scene environments by pre-defining character, script, and scene details.

The content of this page is generated based on existing Markdown specifications and structured diagrams, recommended as the HTML version of project setting documents.
Overview

By setting character appearance, scene standards, prop specifications, and visual coherence rules, the goal of CMS-VFM is to maintain unity in character proportions, lighting styles, wardrobe states, and action continuity in visual generation projects, thereby reducing rework and visual drift.

Structured map of CMS-VFM architecture and visual coherence rules.

4. Visual Coherence Rules

4.1 Shot-to-Shot Consistency

  • Uniform Character Scale: Consistent character size and proportions across different shots
  • Consistent Lighting Direction: Light source direction remains uniform across different shots
  • Consistent Wardrobe State: Wardrobe status remains the same across different shots

4.2 Cross-Scene Continuity

  • Character State Continuity: Cross-scene continuation of scars, makeup, and wardrobe states

4.3 Action Continuity

Ensure character actions are continuous between shots, avoiding chopped or disjointed performances.

4.4 Proportion Consistency

  • Uniform Head-to-Body Ratio: Standard ratio of character head to body
  • Uniform Limb Proportions: Limb length proportion specifications
  • Uniform Prop Size: Consistent prop dimensions across different scenes
  • Phase Proportion Chart: Proportion change table during character growth

4.5 Style Coherence

  • Unified Color Language (Color Script): Project-wide color scheme
  • Unified Lighting Logic (Lighting Bible): Light and shadow performance specifications
  • Unified Material Representation (Material Guide): Material rendering standards

6. Examples & Usage Instructions

It is recommended to use HTML documents as the primary setting format, supporting text, images, videos, and audio content for easy real-time reading and modification by models.

Usage Workflow:

  • Establish a complete CMS-VFM system
  • Develop detailed design specifications
  • Train team members to use specification documents

In the actual process, you should:

  • Read Character Design Sheets and HTML Document Skills every time to avoid shot setting drift
  • Update Environment Design Sheets and character evolution/changes in a timely manner
  • Write documents using HTML markup language, inserting reference materials to ensure models understand the script document content at once
